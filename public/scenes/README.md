# Сцена калабані (процедурна вода)

Калюжа рендериться **процедурно** на `<canvas>` (`src/water/waterRenderer.js`):
намальований фон ями + карта глибини + Canvas2D-вода в перспективі (відбиток неба,
плавне наповнення по рівню води, брижі по кліку, день/ніч). Рівень води й час доби —
**параметри, не файли**, тож одна сцена дає всі проміжні стани без купи асетів.

| Файл                   | Що це | Формат |
|------------------------|-------|--------|
| `puddle-bg.webp`       | денний намальований фон порожньої ями (статична підкладка) | WebP lossy, ~1600px |
| `puddle-bg-night.webp` | нічний фон (зараз — затемнений плейсхолдер з денного; заміни намальованим) | WebP lossy, ~1600px |
| `puddle-map.webp`      | **карта глибини**: яскравість = глибина (біле=найглибше, чорне=поза ямою) | WebP **lossless**, grayscale, ~1000px |
| `altar.webp`           | тло екрана-вівтаря (меню) — опційне | WebP, ~1600px |

**Важливо:**
- `puddle-map.webp` мусить бути **lossless** (або PNG). Рендер трактує яскравість як
  неперервну глибину й порогує її щопікселя — lossy-стиснення дало б терасування берега.
- `puddle-bg.webp` і `puddle-map.webp` мають бути **зареєстровані 1:1** (однакове
  кадрування ями), інакше вода зміститься відносно намальованого краю.
- Якщо фон/карта не завантажились — гра плавно відкочується до процедурної CSS-калюжі.

Замінити арт = поклади файл із тим самим іменем (те саме кадрування). Карту глибини
зручно малювати в ч/б поверх фону (біле в центрі дна, темне до країв).
